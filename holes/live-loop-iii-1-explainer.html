The γ fix, in one picture

Before III.1, the expected leg and realized leg could separate like this: 
expectedG = escrow fold ΔG, e.g. -0.615
enacted wiring = fold_engine RULES result
RULES knows no aif/* patterns -> 0 boxes
coverage-delta-g over 0 boxes -> nil
γ sample absent -> γ holds at prior
The repaired semantics: 
0 boxes + N obligations = measured zero coverage
realizedG = 0.0
signed error = realizedG - expectedG
example: 0.0 - (-0.615) = +0.615
This is deliberately not escrow-as-realized. If realizedG copied expectedG, γ would get zero error by construction and learn nothing. With zero-coverage semantics, the machine sees that the current executor failed to realize the plan and can calibrate accordingly.
