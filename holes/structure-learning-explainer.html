Structure-learning explainer — A4a / R17

What we learned making it real at the pattern grain · siblings: aif-wiring-explainer · aif-r1-r16-pattern-map · pipeline-pattern-cascade. Updated 2026-07-08 (claude-5, with Joe) — verified numbers, honest verbs.

The one-sentence result. Structure learning (forming concepts and scoring their epistemic value from real evidence) is real at the pattern grain — but the working mechanism is embedding-based clustering + Bayesian-Model-Reduction epistemic value (EIG), not naive Bayesian Model Reduction over co-occurrence counts. Naive count-BMR turned out to be miscalibrated at scale; the co-embedded metric space is what makes structure learning tractable. That negative-and-positive, on one real dataset, is the empirical spine of the "AIF and Embeddings" thesis.

What we're aiming for

Active inference has three optimisation levels (after Neacsu 2024, Friston lab): inference (infer states/policies), parametric learning (accumulate Dirichlet counts = Hebbian), and structure learning (reorganise the model — merge look-alike concepts, prune redundant structure). Only the third forms concepts. A4a is the structure-learning node: from accumulated evidence, (a) group units into concepts, (b) score each concept's epistemic value (how much is left to learn), and (c) let a diverse candidate set be re-ranked to prefer what's worth exploring.

;; evidence                concepts             epistemic value        selection
co-occurrence + embedding ──▶ concepts ──▶ EIG (posterior spread) ──▶ ranker (G = risk − λ·EIG)

What happened when we made it real

We stopped trusting the synthetic demo and ran on the real stack. Five findings, each with verified numbers:

1 — The capability-star grain was the wrong shape dead end, informative

The first candidate corpus — the capability star-map (capability → the mission that minted it) — is bijective: each capability is minted once, by one mission. Zero co-occurrence, so BMR had nothing to compare and collapsed all 30 capabilities into 1 concept. Lesson: structure learning needs units that co-occur. We pivoted to the pattern grain, where co-occurrence (missions apply many patterns) and a co-embedding both genuinely exist.

2 — Naive count-BMR is miscalibrated at high dimension the key negative

real corpus: 118 patterns × 94-mission outcome space, 6903 pairs
ALL 6903 pairs accepted (ΔF −6.5 … −33.7). ZERO rejections. → 1 concept.
even two patterns sharing ZERO missions score ΔF ≈ −3.4 (passes the −3 cutoff)

The BMR complexity term scales with the outcome dimension, so at 94 missions every pair looks mergeable — the ΔF ≤ −3 rule is meaningless at scale. Pure Bayesian Model Reduction over raw co-occurrence counts does not work for real structure learning. (A tiny 5-mission case behaves correctly — merges identical, rejects disjoint — so it's specifically a high-dimensional effect. Independently verified.)

3 — The embedding is load-bearing, not enrichment the AIF-and-Embeddings result

On the same data where count-BMR collapses, the existing semilattice constellation clustering (patterns grouped by embedding proximity + citation-overlap — pipeline-pattern-cascade.html, constellations P0–P17) produces coherent, labelled concepts. The metric space is what regularises the sparse high-dimensional counts. Embedding-gated BMR (cosine floor 0.7 + a stricter ΔF cutoff −20) recovers clusters matching the constellations. So the embedding isn't a nod to how we started — it is necessary for structure learning to work here.

The concepts already existed as the constellations. We use them; we do not re-derive them.

4 — The EIG layer, computed over the real constellations the new contribution

The constellations have structure but not epistemic value. BMR supplies it — the posterior spread of each concept's Dirichlet row:

constellation   size   EIG (posterior stddev)   label
P3/P7/P12/P14      1        0.029   singletons — MOST uncertain (least evidence)
P0                 8        0.018   writing / pattern discipline
P8                 7        0.015   agency / coordination / routing
P6                11        0.014   runtime invariants / boot reachability
P1                19        0.011   futon-theory / storage
P13               25        0.010   math proof / rational reconstruction
P2                28        0.009   AIF / operator — MOST settled (most evidence)

EIG scales as 1/√(evidence): concepts we've seen least carry the most epistemic value — the "explore what you know least" signal a ranker needs. (Verified the singleton value by hand: α₀=11.4, RMS of the per-component Dirichlet stddevs = 0.0289.) It is a coarse signal — inverse-evidence — but the right direction.

5 — BMR proposes candidate coarser merges the clustering split a real signal, unvalidated

Does embedding-gated BMR ever see structure the constellations don't have? Checked directly:

tightest merges (cosine ≥ 0.8): all WITHIN one constellation  → pure recovery
working-setting 12-pattern "AIF" cluster spans THREE constellations:
   C2 (core AIF, 7) + C8 (learn-as-you-go, loop-failure-signals, structured-events-only, 3) + C1 (2)
   → a coherent "AIF-and-its-loop" concept the hard clustering split apart
the 8-pattern math-proof cluster: all C13  → pure recovery
also: within-constellation pairs merge 1.55 ΔF more strongly than across (weak corroboration;
      hub patterns like `stop-the-line` bridge constellations and confound it)

So BMR+embedding does propose coarser concepts than the constellations — but at the setting where over-merge creep begins, and with no validation yet that the coarser boundary is principled.

What we can — and can't — claim

The honest verb is recovery + epistemic-quantification + candidate coarser-merges — not "learned structure from scratch." BMR (a) recovers the constellations at fine grain, (b) quantifies their epistemic value (the EIG layer, genuinely new), and (c) proposes candidate coarser merges (e.g. AIF-core + loop-signals across 3 constellations). The coarser merges are a real signal, but they appear near the over-merge threshold and carry no held-out validation, so they are candidates, not confirmed concepts. We do not claim more than the evidence supports.

A clean fact for the roadmap: this all ran on existing co-application data with zero substrate writes — no operator "arm the write" was needed. The only remaining step to feed the ranker is a build (wire {concept → EIG} into the pattern-grain ranker), not an arm.

The next experiment — validate a single coarser merge

The experiment. Take the one coarser merge BMR+embedding proposed that the constellations split — the AIF-12 cluster spanning C2+C8+C1 — and test whether that boundary is principled against a signal independent of the embedding: a held-out split (do the C8/C1 members behave like the C2 core on missions the clustering didn't see?) or a discharge/usage signal (do they get used together in practice?).

What we expect to take away. One of two clean outcomes, either of which is worth having:
  • If the coarser merge validates → it is a genuine learned structure the prior clustering missed. That upgrades the verb from "recovery + candidate" to "structure learning proper," and drops the "recovery" hedge — the strongest form of the A4a claim.
  • If it does not → the honest ceiling is confirmed at "recovery + epistemic-quantification," the constellation granularity is vindicated as correct, and the AIF-12 was over-merge creep — which itself sharpens where the embedding-gate thresholds should sit.
Either way we convert a candidate into a fact — which is the whole discipline this line of work runs on.
