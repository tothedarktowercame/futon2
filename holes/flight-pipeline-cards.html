One flight, in order — the pipeline cards

What ACTUALLY happens, stage by stage, when the War Machine flies one tick. Linear list-of-cards (after futon6's pre-superpod-pipeline-readiness page) — but where that page listed things TO BUILD, these stages are BUILT and marked LIVE / SWAPPABLE / RETIRED. Each card: input → what happens (with the real code location) → output, and a click-through EXAMPLE using real data from the 2026-07-05 11:00Z tick — the first tick in history where a deposited plan was read unattended. Theory + expectations: sequel-notebook.org §5. Diagnosis history: the live-loop explainer trilogy.

1Wake LIVE

input the wall clock; the crontab
happens cron (0 * * * *) launches a one-shot JVM: clojure -M:wm-scheduled (wm_scheduled_run.clj). No server touched; loads source fresh from disk (rulings land at the next tick automatically).
output one judged, gated, traced, logged tick; JVM exits
example — the tick's log line (11:00Z)
2026-07-05T11:04:11Z mode=hermit candidates=145 decision=advance-mission
target=M-first-flights G=8.7112 gates={:abstain-missing-leg 1, :pass 1}
ENACTED=M-bayesian-structure-learning ... realizedG=-0.1666...

2Scan & judge LIVE

input the mission corpus (~145 candidates) + capability/scope state (substrate-2)
happens candidates are filtered to LIVE missions (mission_registry.clj classify-status — the supersession fix, finding 2) and ranked by the 8-term G blend under the current mode (hermit/stop-the-line); the judge emits ranked actions.
output ranked actions: [{:action :advance-mission :target M-… :G …} …]
example — what the filter now catches
M-bayesian-structure-learning status: "SUPERSEDED-AS-MISSION (Campaign recast 2026-06-08)"
old classifier: head token "SUPERSEDED-AS-MISSION" ∉ #{"SUPERSEDED"…} → :unknown → KEPT (ranked top-2 for weeks)
fixed classifier: prefix match → :inactive → filtered from the lane

3Build the circumstance ψ LIVE

input one mission target (e.g. M-diagramprover)
happens cascade_lane.clj mission->psi: if the mission has held-work items (140 mission-linked items in held-work-ledger.edn), build SORRY-GRAIN ψ — WANT: reason/evidence-condition, HUNGRY-FOR: kind, HAVE: re-entry — capped 1KB; else fall back to the banner (title+status), else the id-stem. Cache invalidatable via clear-psi-cache!.
output ψ: a short text stating want / hungry-for / have
example — a real sorry-grain ψ (autoclock-in, from the golden deposit)
WANT: clock-in : (agent, session, target) -> DurableLineage. reconstitute output —
{missions [{:target :agents :sessions :held-count}]} surviving teardown.
HAVE: agent nodes (agent:<id>); canonical mission nodes …; clock/clocked-on hyperedge
(the durable clock record); clock_store ns (RAM per-agent session clocks +
set-dispatch-mission!); agent-chat.el …; held/on-mission hyperedges (109).

known gap (ψ-v3 pending): ψ describes what the mission WANTS, never how the process is FAILING — inject the futility line ("selected N ticks, 0 wins") and the process-coherence seed patterns retrieve at rank 1 (seed-check, finding 6).

4Condense the cascade (the argument) LIVE budget=constant

input ψ (text)
happens cascade_serve.py → construct_cascade(ψ): embed ψ (MiniLM), rank the ~1,180-pattern library by relevance, then phylogeny-greedy coverage-saturation assembles the best small ARGUMENT — seeds prior-floored (0.647 nats), budget 20 as the parsimony ceiling. Scored F = accuracy − λ·complexity.
output the cascade: patterns + rel + shown window + F
example — the aif-head cascade (deposit 004, real run)
size 21 (shown 20, budget 20), F = +8.722
top: ants/baseline-cyber-ant (rel .679) · futon-theory/structural-tension-as-observation (.613)
aif/candidate-pattern-action-space (.526) · aif/structured-observation-vector (.502)
gauntlet/aif-as-environment-not-instruction (.565) · aif/belief-state-operational-hypotheses (.501) …

counter-example (library gap, finding 3): diagramprover's well-formed ψ → size 1, F −0.443 — the library has no proof-search vocabulary; remedy = author patterns via the seeds pipeline, not enrich ψ.

5Price the plan (ΔG reconciliation) LIVE

input the mission + its cascade
happens close_loop.clj act-gate-from-lane-entry consults plan sources IN ORDER: (1) rollout valuation — project-policy prices a move if one exists (the depth>1 SEARCH is RETIRED — level error, see notebook §5.2); (2) classical fold — RETIRED as a route (*classical-fold-dG?* false; rule-table prices weren't staked beliefs); (3) LLM fold via ESCROW — if a deposit matches the mission, reconstruct the deposit's own circumstance, verify the prompt-sha pin + re-assert stored ΔG, and REPLAY the plan.
output ΔG + provenance (:delta-G/source), or nil (honest abstention)
example — the first natural escrow read (11:00Z tick, trace verbatim)
:delta-G-source :fold-escrow, :gate-delta-G -0.7   ; ft-bayesian-structure-learning-003
; ΔG = −(boxes/(boxes+holes)) = −(14/20) = −0.7 — hand-derivable from the deposit
; canon-fingerprint, no deposit → ΔG nil → :abstain-missing-leg (the ruling working)

6The act-gate (∧) LIVE

input ΔF (the argument's quality) and ΔG (the plan's promise)
happens pass ⇔ ΔF > 0 ∧ ΔG < 0. Rhetoric without a plan abstains; a plan without an argument abstains.
output verdict per lane entry: :pass / :fail / :abstain-missing-leg
example — both branches on one tick
M-bayesian-structure-learning: ΔF +1.049 ∧ ΔG −0.7 (escrow) → :pass
M-canon-fingerprint-store:     ΔF −0.874 ∧ ΔG nil          → :abstain-missing-leg
; and the gate REFUSED our own golden deposit earlier (ΔF −0.69) — no favoritism

7Select & enact LIVE γ coasting

input ranked actions + gate verdicts + γ (per-lane precision)
happens selection takes the G-argmin (WM-I4 abstains on flat fields); enactment is CONSENT-GATED and artifact-only — the deterministic executor realizes the passed plan and records realized ΔG. γ should weight selection by which lanes' plans PAY — currently coasting (its expected-G leg was the classical fold; diet decision with the operator).
output the ENACTED line + realized ΔG
example — expected vs realized (11:00Z)
expected (deposit): -0.7      realized (executor): -0.1666…
; the gap is the learning signal γ SHOULD eat (3b sim: paying lane γ→1.26, futile γ→0.50)

8Trace, observe thyself, learn LIVE learning half pending

input everything the tick did
happens write-trace! appends the daily trace (γ-state reads BACK from it — witness records stay quarantined); the futility observable (wm_futility.clj summary) computes strict per-lane 0-for-N; the nag bulletin fires on thresholded futility (dry-run). Library growth (new patterns via the seeds pipeline; co-app accrual) closes R17 — the miner does not yet scan cascade usage.
output the trace record; queryable futility; (dry-run) operator bulletins
example — the futility board (real, 733 ticks)
address-sorry/pudding-g1-arrow-witness-binding  0-for-306
learn-action-class                              0-for-141
advance-mission/M-first-flights                 0-for-51
; 11 lanes, ALL 0-for-N under strict credit: the selected action has never
; passed its own gate. γ, once fed, would learn to doubt exactly this.

That is one flight. The loop in one line: wake → scan/judge → ψ → cascade (F) → plan (ΔG) → gate (∧) → select/enact → trace/learn.
