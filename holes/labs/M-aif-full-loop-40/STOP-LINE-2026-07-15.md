# Stop-the-line record — repeated unconstructible policy

**Declared:** 2026-07-15  
**Cohort:** `:wm-outer-loop-40-v1`  
**Scheduler status:** disabled  
**Active cohort boundary:** `attempt-001` closed; 39 opportunities remaining

## Finding

The first production opportunity (`attempt-001`) completed as a grounded
change.  Attempts 2–22 and 24 selected the same meta-action:

```clojure
{:type :learn-action-class
 :target-class :fire-pattern}
```

The policy layer admitted this action, but the coding full-loop constructor
could not construct it.  Each opportunity therefore closed as `:no-selection`
before agent dispatch.  Attempt 23 closed as `:agent-unavailable` because the
configured reviewer was already invoking on another Agency job; no usage-limit
or rate-limit error was observed.

The repeated failure exposed two coupled defects:

1. **Policy-support mismatch.**  The production coding surface can select an
   action for which it has no construction/actuation path.
2. **Failure-memory gap.**  A post-selection construction failure closes as
   `:no-selection`; it neither opens a stop-the-line obligation nor prevents
   the habit prior from reinforcing and repeating the failed action.

futility index was invalidated so that it rebuilds from the rewound trace.

## Re-enable gate

The scheduled line must remain disabled until all of the following hold:

1. Production policy support excludes actions without a construction and
   actuation path, or provides a genuine typed path for them; selection and
   execution must agree by construction.
2. Any post-selection construction failure creates durable stop-the-line
   memory and cannot be reinforced by the habit prior as a successful habit.
3. Tests reproduce the `:learn-action-class/:fire-pattern` failure and prove
   both the support-boundary and failure-memory repairs.
4. Repository-required static checks and relevant tests pass.
5. A full production-shaped canary reaches independent review and grounded
   adjudication without special-casing the selected action.
6. A distinct reviewer approves the repair and canary evidence.
7. The resumed attempt records a new semantic epoch.

The old `repair-obligation` schema covers independent-review failures only; it
cannot honestly represent this system-level failure.  Extending that schema is
part of the required structural repair, rather than encoding this incident as
a fake reviewer rejection.
