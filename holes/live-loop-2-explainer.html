What this audit does NOT show (the teeth)

  1. The live read was a witnessed lane-entry evaluation in the serving JVM — not a scheduled tick. Honest reason, from the runner's own audit: the scheduled caller (enact.clj:91) never injects the escrow turn-fn, and the live lane's ψ is still banner-grain, so the sha pin would rightly abstain against any prompt the lane can currently build. That is the pin working — and it means ledger §10's falsifier ("post-S2 ticks still 0-for-N") has not yet had its real test. Named follow-on: wire the scheduled caller + sorry-grain lane ψ.
  2. There are TWO escrow seams. An older :llm-escrow branch in the scheduled path reads un-pinned wiring EDN with no arming, no sha, no ΔG re-assert — currently empty but reachable: a consent-bypass-shaped hole. Deprecation recommended before the caller wiring lands.
  3. γ has learned nothing yet. The 3-series is unlocked, not built; the lane still cannot lose precision by failing to pay. One crossing is an existence proof, not a habit.
  4. Two baseline re-pins in one day (F +0.046→+0.051; +1.461→+1.623), both from the seed-registration prior shift — benign, mechanistically explained, and both caught by controls narrower claims had missed. Standing lesson, now in the ledger: "identical" claims must name the exact pipeline stage they were checked at.
  5. The witness trace lives in a quarantined directory on purpose (data/wm-trace-escrow-witness/): the WM reads γ-state back from its own daily trace, and a witness record with default precision could have silently reset learned γ. The hazard was found by reading the read-back path before writing — barely.
